## 연습문제

1. (______) 안에 알맞은 용어는? -- 개체
현실 세계의 데이터는 사람의 오감으로 인지할 수 있는 실체로 둘 이상의 특성으로 구성된 (______)로표현된다.

2. (______) 안에 알맞은 용어는? -- 개체 타입
개넘 세계의 데이터는 개체의 의미로부터 인식된 개념으로， 둘 이상의 속성으로 구성된 (______)으로표현된다.

3. 현실 세계의 직장인이 속한 세계에 따라 구분되는 구성 요소를 5개 이상 작성하시오.

| 구분 | 구성요소 |
|-----|----------|
| 현실 세계의 직장인 | 이름, 나이, 주소, 전화번호, 직장, 직급 |
| 회사라는 개념 세계의 직장인 | 사번, 이름, 부서, 직급, 입사일, 급여 |
| 자동차 보험사라는 개념 세계의 직장인 | 설계사번호, 이름, 소속지점, 담당고객, 판매실적, 자격등급 |
| 회사라는 개념 세계와 대응하는 컴퓨터 세계의 직장인 | 사번필드, 이름필드, 부서코드, 직급코드, 급여 컬럼 |

4. 데이터 모델령의 각단계와설명을바르게 연결하시오.
  1. 개념적스카마모텔링 -- c
  2. 논리적스키마모델링 -- a
  3. 물리적스키마모델렁 -- b
    a. 개념적 모델링 결과인 개념적 구조를 DBMS가이해할 수 있는 논리적 구조， 즉 데이터 모델로 변환하는과정
    b. 디스크에 데이터가 저장될 수 있도록 논리적 구조를 물리적 구조로 변환하는 과정
    c. 현실 세계에서 인석한 데이터를 추상화를 통해서 개념 세계의 추상적 개념으로 표현하는 과정

5. 데이터 모텔링을 개념적 모댈링， 논리적 모댈링， 물리적 모댈링으로 구분해서 수행히는 이유를 설명하시오. 
-- 현실 세계의 복잡한 정보를 한 번에 DB에 저장하기 어렵기 때문에, 먼저 사람이 이해하기 쉬운 개념적 구조로 추상화하고, 그 다음 DBMS가 이해할 수 있는 논리 구조로 바꾸고, 마지막으로 저장 장치에 맞는 물리 구조로 변환한다. 이렇게 나눠서 설계해야 체계적이고, 사용자 관점과 시스템 관점을 구분할 수 있으며, 변경이나 구현도 쉬워지기 때문.

6. 개념적 구조， 즉 개념적 모댈링 결과를 DBMS가 직접 이해할수 없는 이유를 설명하시오.
-- 개념적 구조는 현실 세계를 사람 중심으로 추상화한 표현이라서 개체 타입, 속성, 관계처럼 의미 중심으로 기술. 반면, DBMS는 테이블, 레코드 타입, 키, 데이터 타입 같은 논리적 구조를 필요로 하므로, 개념적 구조를 바로 처리할 수 없습니다.

  2. 요구 수집 및 분석 - DBMS 선정 - 논리적 설계 - 개념적 설계 - 물리적 설계 - 구현 및 태스트
  3. 요구 수집 및 분석 - 개념적 설계 - DBMS 선정 - 논리적 설계 - 물리적 설계 - 구현 및 테스트
  4. 요구 수집 및 분석 - 개념적 설계 - 논리적 설계 - DBMS 선정 - 물리적 설계 - 구현 및 태스트

10. 다음 중 DB 설계 전략에 대한 설명이 잘못된 것은?
  1. DB의 내용과 구조에 치중해서 설계히는 방법은 데이터 중심 설계다.
  2. 데이터의 처리와응용에 치중해서 설계하는 방법은 처리 중심 설계다.
  3. 데이터 중심 설계와 처리 중심 설계 가운데 하나를 선택해서 진행하는 것이 효율적이다.
  4. 데이터 중심 설계와 처리 중심 설계를 병행해서 진행히는 것이 효율적이다.

11. 다음 중 데이터베이스 설계 시 고려 사항에 대한 설명이 잘못된 것은?
  1. 보안: 불법적인 접근(데이터의 변경 ， 손실， 노출 등)에 대해서 DB를 보호할 수 있어야한다.
  2. 확장성:시스템운영을일시중단하고새로운데이터나응용프로그램을 추가할수있어야한다.
  3. 일관성: 저장된 데이터 값간또는특정 질의에 대한응답들 간에 상호 모순 없이 일치해야한다.
  4. 무결성 :DB에 저장된 데이터 값은제약조건을항상만족해야한다.
